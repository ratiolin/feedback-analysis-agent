# 简历描述候选

## 客户反馈结构化分析 Agent｜个人项目

设计并实现面向项目协作 SaaS 的客户反馈工作流，支持工单导入、结构化分类、确定性责任路由、重复问题聚类、候选 SOP 和可追溯周报。采用“LLM 建议 + Schema/证据硬门 + 人工确认”边界：模型只输出逐字 quote，由服务端定位 offset；根因假设不进入周报事实结论，候选 SOP 不自动进入正式知识库。使用 FastAPI、Next.js、Dify、DeepSeek、BGE、PostgreSQL 与 Docker Compose 完成公网演示。

四个 Dify 工作流已发布并完成真实 API 回放。V7 合成锁定集 N=60 中，Schema 与 quote 定位成功率均为 100%，问题类型 Macro-F1 0.846、产品区域 Macro-F1 0.963、责任路由一致率 85.0%、高风险升级召回率 100%；重复识别精确率 84.2%、召回率 53.3%、聚类纯度 96.7%。公开混淆矩阵、错误合并/拆分案例及 V5/V6 失败历史，并以独立 promotion record 固化作品集演示配置。

## 商品上架运营自动化｜补充项目

实现“模拟商品 CSV → 异常校验 → Dify 候选文案 → 影刀编排 → Odoo ERP 写入 → Metabase 看板”的本地沙箱。确定性规则控制准入，影刀 6.0.30 通过 Odoo XML-RPC 按 SKU 幂等写入，API 使用稳定 operation key 回传审计结果，Metabase 以只读账号展示当前状态与历史操作。真实沙箱回放 30 条合成商品：10 条异常被门禁阻止，20 条由已发布 Dify 工作流生成文案并成功写入 Odoo，当前 ERP 写入错误 0 条；保留 23 次开发调试失败尝试作为审计历史。

## 表述边界

以上均为合成作品集与机制验证。可以描述平台实操、真实 API 回放、锁定评测及失败历史；不得写成真实生产上线、独立人工审计、真实效率提升或销售收益。
