# 评测口径与证据边界

## 数据切分

- 总量 N=240。
- 开发/演示集 N=180，用于提示词、规则和聚类阈值开发。
- 锁定人工校验集 N=60，不得用于调整提示词或阈值。
- 60 条锁定集仅用于合成场景机制评估，不代表真实业务分布。

## 展示指标

面试展示页优先使用：

- 问题类型与产品区域混淆矩阵。
- 每类 precision、recall、F1 和 support。
- 重复问题匹配精确率与召回率。
- 聚类纯度。
- 错误合并与错误拆分案例。
- 代表工单可追溯率。

B³ F1 和阈值搜索只进入技术报告。

## 当前事实

当前公开评测来自 `demo_rules` 故障兜底，不是 DeepSeek/Dify 质量结果。它有意保留低分类得分，防止静态页面把兜底规则包装成模型能力。权威数值见 `artifacts/evaluation/evaluation.json`。
